028/**
029 * REST request body annotation.
030 *
031 * <p>
032 * Identifies a POJO to be used as the body of an HTTP request.
033 *
034 * <p>
035 * Can be used in the following locations:
036 * <ul>
037 *    <li>Java method arguments and argument-types of client-side <ja>@Remoteable</ja>-annotated REST interface proxies.
038 *    <li>Java method arguments and argument-types of server-side <ja>@RestMethod</ja>-annotated REST Java methods.
039 * </ul>
040 *
041 * <h5 class='topic'>Server-side REST</h5>
042 *
043 * <p>
044 * On server-side REST, this annotation can be applied to method parameters or parameter classes to identify them as the body of an HTTP request.
045 *
046 * <h5 class='section'>Examples:</h5>
047 * <p class='bcode w800'>
048 *    <jc>// Used on parameter</jc>
049 *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>,path=<js>"/pets"</js>)
050 *    <jk>public void</jk> addPet(<ja>@Body</ja> Pet pet) {...}
051 * </p>
052 * <p class='bcode w800'>
053 *    <jc>// Used on class</jc>
054 *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>,path=<js>"/pets"</js>)
055 *    <jk>public void</jk> addPet(Pet pet) {...}
056 *
057 *    <ja>@Body</ja>
058 *    <jk>public class</jk> Pet {...}
059 * </p>
060 *
061 * <p>
062 * This is functionally equivalent to the following code...
063 * <p class='bcode w800'>
064 *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>,path=<js>"/pets"</js>)
065 *    <jk>public void</jk> addPet(RestRequest req) {
066 *       Pet pet = req.getBody().asType(Pet.<jk>class</jk>);
067 *       ...
068 *    }
069 * </p>
070 *
071 * <p>
072 * Any of the following types can be used for the parameter or POJO class (matched in the specified order):
073 * <ol class='spaced-list'>
074 *    <li>
075 *       {@link Reader}
076 *       <br><ja>@Body</ja> annotation is optional.
077 *       <br><code>Content-Type</code> is ignored.
078 *    <li>
079 *       {@link InputStream}
080 *       <br><ja>@Body</ja> annotation is optional.
081 *       <br><code>Content-Type</code> is ignored.
082 *    <li>
083 *       Any <a class='doclink' href='../../../../../overview-summary.html#juneau-marshall.PojoCategories'>Parsable POJO</a> type.
084 *       <br><code>Content-Type</code> is required to identify correct parser.
085 *    <li>
086 *       Objects convertible from {@link Reader} by having one of the following non-deprecated methods:
087 *       <ul>
088 *          <li><code><jk>public</jk> T(Reader in) {...}</code>
089 *          <li><code><jk>public static</jk> T <jsm>create</jsm>(Reader in) {...}</code>
090 *          <li><code><jk>public static</jk> T <jsm>fromReader</jsm>(Reader in) {...}</code>
091 *       </ul>
092 *       <code>Content-Type</code> must not be present or match an existing parser so that it's not parsed as a POJO.
093 *    <li>
094 *       Objects convertible from {@link InputStream} by having one of the following non-deprecated methods:
095 *       <ul>
096 *          <li><code><jk>public</jk> T(InputStream in) {...}</code>
097 *          <li><code><jk>public static</jk> T <jsm>create</jsm>(InputStream in) {...}</code>
098 *          <li><code><jk>public static</jk> T <jsm>fromInputStream</jsm>(InputStream in) {...}</code>
099 *       </ul>
100 *       <code>Content-Type</code> must not be present or match an existing parser so that it's not parsed as a POJO.
101 *    <li>
102 *       Objects convertible from data types inferred from Swagger schema annotations using the registered {@link OpenApiPartParser}.
103 *       <br>
104 *       <code>Content-Type</code> must not be present or match an existing parser so that it's not parsed as a POJO.
105 * </ol>
106 *
107 * <p>
108 * Also used to populate the auto-generated Swagger documentation.
109 *
110 * <h5 class='section'>Examples:</h5>
111 * <p class='bcode w800'>
112 *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>,path=<js>"/pets"</js>)
113 *    <jk>public void</jk> addPet(Pet pet) {...}
114 *
115 *    <ja>@Body</ja>(
116 *       description=<js>"Pet object to add to the store"</js>,
117 *       required=<jk>true</jk>,
118 *       example=<js>"{name:'Doggie',price:9.99,species:'Dog',tags:['friendly','cute']}"</js>
119 *    )
120 *    <jk>public class</jk> Pet {...}
121 * </p>
122 *
123 * <p>
124 * Swagger documentation values are coalesced from multiple sources in the following order of precedence:
125 * <ol>
126 *    <li><ja>@Body</ja> annotation on parameter.
127 *    <li><ja>@Body</ja> annotation on parameter class.
128 *    <li><ja>@Body</ja> annotation on parent classes and interfaces.
129 *    <li><ja>@MethodSwagger(value)</ja> annotation.
130 *    <li>Localized resource bundle property <js>"[method-name].produces"</js>.
131 *    <li><ja>@ResourceSwagger(value)</ja> annotation.
132 *    <li>Localized classpath resource file <js>"[enclosing-class].[simple-class-name]_[locale].json"</js> (if it's an inner or member class).
133 *    <li>Default classpath resource file <js>"[enclosing-class].[simple-class-name].json"</js> (if it's an inner or member class).
134 *    <li>Localized classpath resource file <js>"[simple-class-name]_[locale].json"</js>.
135 *    <li>Default classpath resource file <js>"[simple-class-name].json"</js>.
136 * </ol>
137 *
138 * <h5 class='section'>See Also:</h5>
139 * <ul>
140 *    <li class='link'><a class="doclink" href="../../../../../overview-summary.html#juneau-rest-server.HttpPartAnnotations.Body">Overview &gt; juneau-rest-server &gt; @Body</a>
141 *    <li class='link'><a class="doclink" href="../../../../../overview-summary.html#juneau-rest-server.Swagger">Overview &gt; juneau-rest-server &gt; OPTIONS pages and Swagger</a>
142 *    <li class='link'><a class="doclink" href="https://swagger.io/specification/v2/#parameterObject">Swagger Specification &gt; Parameter Object</a>
143 * </ul>
144 *
145 * <h5 class='topic'>Client-side REST</h5>
146 *
147 * Annotation applied to Java method arguments of interface proxies to denote that they are the HTTP body of the request.
148 *
149 * <h5 class='section'>Examples:</h5>
150 * <p class='bcode w800'>
151 *    <jc>// Used on parameter</jc>
152 *    <ja>@Remoteable</ja>(path=<js>"/petstore"</js>)
153 *    <jk>public interface</jk> PetStore {
154 *
155 *       <ja>@RemoteMethod</ja>(path=<js>"/pets"</js>)
156 *       String addPet(<ja>@Body</ja> Pet pet);
157 *    }
158 * </p>
159 * <p class='bcode w800'>
160 *    <jc>// Used on class</jc>
161 *    <ja>@Remoteable</ja>(path=<js>"/petstore"</js>)
162 *    <jk>public interface</jk> PetStore {
163 *
164 *       <ja>@RemoteMethod</ja>(path=<js>"/pets"</js>)
165 *       String addPet(Pet pet);
166 *    }
167 *
168 *    <ja>@Body</ja>
169 *    <jk>public class</jk> Pet {...}
170 * </p>
171 *
172 * <p>
173 * The annotation can also be applied to a bean property field or getter when the argument is annotated with
174 * {@link RequestBean @RequestBean}:
175 *
176 * <h5 class='section'>Example:</h5>
177 * <p class='bcode w800'>
178 *    <ja>@Remoteable</ja>(path=<js>"/myproxy"</js>)
179 *    <jk>public interface</jk> MyProxy {
180 *
181 *       <ja>@RemoteMethod</ja>(path=<js>"/mymethod"</js>)
182 *       String myProxyMethod(<ja>@RequestBean</ja> MyRequestBean bean);
183 *    }
184 *
185 *    <jk>public interface</jk> MyRequestBean {
186 *       <ja>@Body</ja>
187 *       MyPojo getMyPojo();
188 *    }
189 * </p>
190 *
191 * <p>
192 * The argument can be any of the following types:
193 * <ul class='spaced-list'>
194 *    <li>
195 *       Any serializable POJO - Converted to text using the {@link Serializer} registered with the
196 *       <code>RestClient</code>.
197 *    <li>
198 *       {@link Reader} - Raw contents of {@code Reader} will be serialized to remote resource.
199 *    <li>
200 *       {@link InputStream} - Raw contents of {@code InputStream} will be serialized to remote resource.
201 *    <li>
202 *       <code>HttpEntity</code> - Bypass Juneau serialization and pass HttpEntity directly to HttpClient.
203 *    <li>
204 *       <code>NameValuePairs</code> - Converted to a URL-encoded FORM post.
205 * </ul>
206 *
207 * <p>
208 * OpenAPI schema based serialization can be used by specifying a value for the {@link #partSerializer()} annotation.
209 *
210 * <p class='bcode w800'>
211 *    <ja>@RemoteMethod</ja>(path=<js>"/comma-delimited-pipe-delimited-ints"</js>)
212 *    String addCommaDelimitedPipeDelimitedInts(
213 *       <ja>@Body</ja>(
214 *          serializer=OpenApiPartSerializer.<jk>class</jk>,
215 *          schema=<ja>@Schema</ja>(
216 *             type=<js>"array"</js>,
217 *             collectionFormat=<js>"pipes"</js>,
218 *             items=<ja>@Items</ja>(
219 *                type=<js>"array"</js>
220 *                items=<ja>@SubItems</ja>(
221 *                   type=<js>"int32"</js>,
222 *                   <jc>// Auto-validates on client side!</jc>
223 *                   minimum=<js>"0"</js>,
224 *                   maximum=<js>"64"</js>
225 *                )
226 *             )
227 *          )
228 *       )
229 *       <jk>int</jk>[][] input
230 *    );
231 * </p>
232 *
233 * <p>
234 * When using OpenAPI serialization, the argument can be any data type specified in {@link OpenApiPartSerializer}.
235 *
236 * <h5 class='section'>See Also:</h5>
237 * <ul class='doctree'>
238 *    <li class='link'><a class='doclink' href='../../../../../overview-summary.html#juneau-rest-client.HttpPartAnnotations.Body'>Overview &gt; juneau-rest-client &gt; Interface Proxies Against 3rd-party REST Interfaces &gt; Body</a>
239 * </ul>
240 */
241@Documented
242@Target({PARAMETER,FIELD,METHOD,TYPE})
243@Retention(RUNTIME)
244@Inherited
245public @interface Body {
246
247   /**
248    * Specifies the {@link HttpPartSerializer} class used for serializing values to strings when using schema-based HTTP part serializing.
249    *
250    * <p>
251    * Overrides for this part the part serializer defined on the REST client which by default is {@link OpenApiPartSerializer}.
252    */
253   Class<? extends HttpPartSerializer> partSerializer() default HttpPartSerializer.Null.class;
254
255   /**
256    * Specifies the {@link HttpPartParser} class used for parsing strings to values when using when using schema-based HTTP part parsing.
257    *
258    * <p>
259    * Overrides for this part the part parser defined on the REST resource which by default is {@link OpenApiPartParser}.
260    */
261   Class<? extends HttpPartParser> partParser() default HttpPartParser.Null.class;
262
263   /**
264    * Specifies whether a part serializer should be used for serializing this value.
265    *
266    * <p>
267    * If <jk>false</jk>, then it indicates that normal Juneau serializers (e.g. {@link JsonSerializer}) should be used for this part.
268    */
269   public boolean usePartSerializer() default false;
270
271   /**
272    * Specifies whether a part parser should be used for parsing this value.
273    *
274    * <p>
275    * If <jk>false</jk>, then it indicates that normal Juneau parsers (e.g. {@link JsonParser}) should be used for this part.
276    */
277   public boolean usePartParser() default false;

283   /**
284    * <mk>description</mk> field of the Swagger <a class="doclink" href="https://swagger.io/specification/v2/#parameterObject">Parameter</a> object.
285    *
286    * <p>
287    * A brief description of the body. This could contain examples of use.
288    *
289    * <h5 class='section'>Examples:</h5>
290    * <p class='bcode w800'>
291    *    <jc>// Used on parameter</jc>
292    *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>)
293    *    <jk>public void</jk> addPet(
294    *       <ja>@Body</ja>(description=<js>"Pet object to add to the store"</js>) Pet input
295    *    ) {...}
296    * </p>
297    * <p class='bcode w800'>
298    *    <jc>// Used on class</jc>
299    *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>)
300    *    <jk>public void</jk> addPet(Pet input) {...}
301    *
302    *    <ja>@Body</ja>(description=<js>"Pet object to add to the store"</js>)
303    *    <jk>public class</jk> Pet {...}
304    * </p>
305    *
306    * <h5 class='section'>Used for:</h5>
307    * <ul class='spaced-list'>
308    *    <li>
309    *       Server-side generated Swagger documentation.
310    * </ul>
311    *
312    * <h5 class='section'>Notes:</h5>
313    * <ul class='spaced-list'>
314    *    <li>
315    *       The format is plain text.
316    *       <br>Multiple lines are concatenated with newlines.
317    *    <li>
318    *       Supports <a class="doclink" href="../../../../../overview-summary.html#DefaultRestSvlVariables">initialization-time and request-time variables</a>
319    *       (e.g. <js>"$L{my.localized.variable}"</js>).
320    * </ul>
321    */
322   String[] description() default {};
323
324   /**
325    * <mk>required</mk> field of the Swagger <a class="doclink" href="https://swagger.io/specification/v2/#parameterObject">Parameter</a> object.
326    *
327    * <p>
328    * Determines whether the body is mandatory.
329    *
330    * <p>
331    * If validation fails during serialization or parsing, the part serializer/parser will throw a {@link SchemaValidationException}.
332    * <br>On the client-side, this gets converted to a <code>RestCallException</code> which is thrown before the connection is made.
333    * <br>On the server-side, this gets converted to a <code>BadRequest</code> (400).
334    *
335    * <h5 class='section'>Examples:</h5>
336    * <p class='bcode w800'>
337    *    <jc>// Used on parameter</jc>
338    *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>)
339    *    <jk>public void</jk> addPet(
340    *       <ja>@Body</ja>(required=<jk>true</jk>) Pet input
341    *    ) {...}
342    * </p>
343    * <p class='bcode w800'>
344    *    <jc>// Used on class</jc>
345    *    <ja>@RestMethod</ja>(name=<jsf>POST</jsf>)
346    *    <jk>public void</jk> addPet(Pet input) {...}
347    *
348    *    <ja>@Body</ja>(required=<jk>true</jk>)
349    *    <jk>public class</jk> Pet {...}
350    * </p>
351    *
352    * <h5 class='section'>Used for:</h5>
353    * <ul class='spaced-list'>
354    *    <li>
355    *       Server-side schema-based parsing validation.
356    *    <li>
357    *       Server-side generated Swagger documentation.
358    *    <li>
359    *       Client-side schema-based serializing validation.
360    * </ul>
361    *
362    * <h5 class='section'>Notes:</h5>
363    * <ul class='spaced-list'>
364    *    <li>
365    *       Supports <a class="doclink" href="../../../../../overview-summary.html#DefaultRestSvlVariables">initialization-time and request-time variables</a>
366    *       (e.g. <js>"$L{my.localized.variable}"</js>).
367    * </ul>
368    */
369   boolean required() default false;

377    *
378    * <p>
379    * The schema defining the type used for the body parameter.
380    *
381    * <p>
382    * This is a required attribute per the swagger definition.
383    * However, if not explicitly specified, the value will be auto-generated using {@link JsonSchemaSerializer}.
384    *
385    * <h5 class='section'>Used for:</h5>
386    * <ul class='spaced-list'>
387    *    <li>
388    *       Server-side schema-based parsing and parsing validation.
389    *    <li>
390    *       Server-side generated Swagger documentation.
391    *    <li>
392    *       Client-side schema-based serializing and serializing validation.
393    * </ul>
394    *
395    * <h5 class='section'>Notes:</h5>
396    * <ul class='spaced-list'>
397    *    <li>
398    *       Supports <a class="doclink" href="../../../../../overview-summary.html#DefaultRestSvlVariables">initialization-time and request-time variables</a>
399    *       (e.g. <js>"$L{my.localized.variable}"</js>).
400    * </ul>
401    */
402   Schema schema() default @Schema;

408   /**
409    * A serialized example of the body of a request.
410    *
411    * <p>
412    * This is the <a class='doclink' href='../../../../../overview-summary.html#juneau-marshall.JsonDetails.SimplifiedJson'>Simplified JSON</a> of an example of the body.
413    *
414    * <p>
415    * This value is converted to a POJO and then serialized to all the registered serializers on the REST method to produce examples for all
416    * supported language types.
417    * <br>These values are then used to automatically populate the {@link #examples} field.
418    *
419    * <h5 class='section'>Example:</h5>
420    * <p class='bcode w800'>
421    *    <jc>// A JSON representation of a PetCreate object.</jc>
422    *    <ja>@Body</ja>(
423    *       example=<js>"{name:'Doggie',price:9.99,species:'Dog',tags:['friendly','cute']}"</js>
424    *    )
425    * </p>
426    * <p>
427    * <img class='bordered' src='doc-files/Body_Example.png' style='width:860px'>
428    *
429    * <p>
430    * There are several other options for defining this example:
431    * <ul class='spaced-list'>
432    *    <li>
433    *       Defining an <js>"x-example"</js> field in the inherited Swagger JSON body field (classpath file or <code><ja>@ResourceSwagger</ja>(value)</code>/<code><ja>@MethodSwagger</ja>(value)</code>).
434    *    <li>
435    *       Defining an <js>"x-example"</js> field in the Swagger Schema Object for the body (including referenced <js>"$ref"</js> schemas).
436    *    <li>
437    *       Allowing Juneau to auto-generate a code example.
438    * </ul>
439    *
440    * <p>
441    * The latter is important because Juneau also supports auto-generation of JSON-Schema from POJO classes using {@link JsonSchemaSerializer} which has several of it's own
442    * options for auto-detecting and calculation POJO examples.
443    *
444    * <p>
445    * In particular, examples can be defined via static methods, fields, and annotations on the classes themselves.
446    *
447    * <p class='bcode w800'>
448    *    <jc>// Annotation on class.</jc>
449    *    <ja>@Example</ja>(<js>"{name:'Doggie',price:9.99,species:'Dog',tags:['friendly','cute']}"</js>)
450    *    <jk>public class</jk> PetCreate {
451    *       ...
452    *    }
453    * </p>
454    * <p class='bcode w800'>
455    *    <jc>// Annotation on static method.</jc>
456    *    <jk>public class</jk> PetCreate {
457    *
458    *       <ja>@Example</ja>
459    *       <jk>public static</jk> PetCreate <jsm>sample</jsm>() {
460    *          <jk>return new</jk> PetCreate(<js>"Doggie"</js>, 9.99f, <js>"Dog"</js>, <jk>new</jk> String[] {<js>"friendly"</js>,<js>"cute"</js>});
461    *       }
462    *    }
463    * </p>
464    * <p class='bcode w800'>
465    *    <jc>// Static method with specific name 'example'.</jc>
466    *    <jk>public class</jk> PetCreate {
467    *
468    *       <jk>public static</jk> PetCreate <jsm>example</jsm>() {
469    *          <jk>return new</jk> PetCreate(<js>"Doggie"</js>, 9.99f, <js>"Dog"</js>, <jk>new</jk> String[] {<js>"friendly"</js>,<js>"cute"</js>});
470    *       }
471    *    }
472    * </p>
473    * <p class='bcode w800'>
474    *    <jc>// Static field.</jc>
475    *    <jk>public class</jk> PetCreate {
476    *
477    *       <ja>@Example</ja>
478    *       <jk>public static</jk> PetCreate <jsf>EXAMPLE</jsf> = <jk>new</jk> PetCreate(<js>"Doggie"</js>, 9.99f, <js>"Dog"</js>, <jk>new</jk> String[] {<js>"friendly"</js>,<js>"cute"</js>});
479    *    }
480    * </p>
481    *
482    * <p>
483    * Examples can also be specified via generic properties as well using the {@link BeanContext#BEAN_examples} property at either the class or method level.
484    * <p class='bcode w800'>
485    *    <jc>// Examples defined at class level.</jc>
486    *    <ja>@RestResource</ja>(
487    *       properties={
488    *          <ja>@Property</ja>(
489    *             name=<jsf>BEAN_examples</jsf>,
490    *             value=<js>"{'org.apache.juneau.examples.rest.petstore.PetCreate': {name:'Doggie',price:9.99,species:'Dog',tags:['friendly','cute']}}"</js>
491    *          )
492    *       }
493    *    )
494    * </p>
495    *
496    * <h5 class='section'>Used for:</h5>
497    * <ul class='spaced-list'>
498    *    <li>
499    *       Server-side generated Swagger documentation.
500    * </ul>
501    *
502    * <h5 class='section'>See also:</h5>
503    * <ul>
504    *    <li class='ja'>{@link Example}
505    *    <li class='jc'>{@link BeanContext}
506    *    <ul>
507    *       <li class='jf'>{@link BeanContext#BEAN_examples BEAN_examples}
508    *    </ul>
509    *    <li class='jc'>{@link JsonSchemaSerializer}
510    *    <ul>
511    *       <li class='jf'>{@link JsonSchemaSerializer#JSONSCHEMA_addExamplesTo JSONSCHEMA_addExamplesTo}
512    *       <li class='jf'>{@link JsonSchemaSerializer#JSONSCHEMA_allowNestedExamples JSONSCHEMA_allowNestedExamples}
513    *    </ul>
514    * </ul>
515    *
516    * <h5 class='section'>Notes:</h5>
517    * <ul class='spaced-list'>
518    *    <li>
519    *       The format is any <a class='doclink' href='../../../../../overview-summary.html#juneau-marshall.JsonDetails.SimplifiedJson'>Simplified JSON</a> if the object can be converted to a POJO using {@link JsonParser#DEFAULT} or a simple String if the object
520    *       has a schema associated with it meancan be converted from a String.
521    *       <br>Multiple lines are concatenated with newlines.
522    *    <li>
523    *       The format of this object can also be a simple String if the body has a schema associated with it, meaning it's meant to be treated as an HTTP part.
524    *    <li>
525    *       Supports <a class="doclink" href="../../../../../overview-summary.html#DefaultRestSvlVariables">initialization-time and request-time variables</a>
526    *       (e.g. <js>"$L{my.localized.variable}"</js>).
527    * </ul>
528    */
529   String[] example() default {};
